# Onboarding and launching design review

## First run and value
- [ ] The first interactive screen shows real content or value, not a gate or marketing splash.
- [ ] A new user can do something successful before being asked for anything.
- [ ] No mandatory account, survey, or setup blocks access to core value.

## Launch screen
- [ ] Launch screen closely matches the first real screen (minus interactive elements).
- [ ] It is not used as a splash, logo reveal, branding moment, or onboarding slide.
- [ ] The app launches in the current device orientation where applicable.

## Onboarding flow
- [ ] Onboarding is justified — the interface is not self-explanatory without it.
- [ ] It is short, fast, and skippable, with a visible skip or dismiss control.
- [ ] It is not shown again to returning users.
- [ ] Instruction is taught in context (learning by doing), not as a screenshot carousel.

## Sign-in and accounts
- [ ] Sign-in is deferred until an action genuinely needs an account.
- [ ] People can explore and create or save locally before signing in.
- [ ] Sign in with Apple and AutoFill / pre-fill reduce account friction to a few taps.

## Permissions
- [ ] Each permission is requested at point of use, not at launch.
- [ ] A short in-context rationale precedes each system prompt.
- [ ] No camera, location, contacts, or notification prompts fire on first open without need.

## Setup and defaults
- [ ] Setup info is sourced from device settings, defaults, and iCloud where possible.
- [ ] Sensible defaults mean most users see no setup step; customization lives in Settings.
- [ ] No in-app license agreement or rating prompt appears in the first session.

## State restoration
- [ ] State is preserved and restored so people resume where they left off.
- [ ] First-run and onboarding flows do not re-run for users who already started.

## Platform fit
- [ ] watchOS pushes meaningful setup to the paired iPhone; first run is near-instant.
- [ ] tvOS onboarding is brief, focus-navigable, and avoids on-screen typing for sign-in.
- [ ] visionOS introduces spatial interactions in context and requests sensing only when needed.
- [ ] macOS opens to a usable window/document and reopens prior session state.
